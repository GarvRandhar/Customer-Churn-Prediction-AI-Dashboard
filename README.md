## Business Objective Metric

The default training objective is:

- **Primary goal:** maximize churn recall
- **Constraint:** churn precision must be at least `0.60`
- **Fallback:** if precision constraint is not achievable, maximize churn F1

This objective is used to auto-select a decision threshold after calibration.

## 🚀 Quick Start

### 1. Clone & Install

```bash
git clone https://github.com/GarvRandhar/Customer-Churn-Prediction-AI-Dashboard.git
cd Customer-Churn-Prediction-AI-Dashboard

# Install Python dependencies
pip install -r requirements.txt

# Install React dependencies
cd client && npm install && cd ..
```

### 2. Run Locally (Development)

**Terminal 1** — Start the Flask backend:
```bash
python app.py
```

**Terminal 2** — Start the React dev server:
```bash
cd client && npm start
```

Open `http://localhost:3000` to view it in the browser. 

*To build for production:*
```bash
cd client && npm run build
# Then visit http://localhost:5001
```

## Model Training Workflow

`train_xgboost.py` now performs:

1. Data cleaning and feature ordering
2. Feature engineering (`avg_monthly_spend`, `active_services_count`)
3. `OneHotEncoder(handle_unknown="ignore")` for categorical robustness
4. Training with the benchmark-winning default XGBoost configuration
5. Isotonic probability calibration
6. Threshold optimization using the business objective metric
7. Export of `model_bundle.pkl` and `model_report.json`

## Benchmark Workflow

Run benchmark comparison:

1. Applies feature engineering (`avg_monthly_spend`, `active_services_count`)
2. Runs stratified CV randomized search per model family
3. Selects best calibration method (`isotonic` vs `sigmoid`) per model
4. Optimizes threshold for the business objective
5. Exports `benchmark_report.json` and `benchmark_leaderboard.md`
